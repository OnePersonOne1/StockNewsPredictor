# 고찰: 모델 유형 × 입력 정제(curation)의 상호작용

## 관찰과 제기된 해석
- **관찰**: 관련성 필터(시장 카테고리만 남김)는 **TF-IDF 를 일관되게 끌어올리나**
  (EXP-C: KOSDAQ h5 0.25→0.45, EXP-E: KOSPI h1 0.30→0.38), **RoBERTa 엔 무이득이고
  market 필터는 오히려 해롭다**(EXP-E: KOSPI h1 0.318→0.226).
- **제기된 해석**: "룰베이스/통계 기반은 입력이 정제돼야 작동하지만, LLM 은 단어 간
  중요도를 스스로 판단·정제하므로 정제의 중요도가 덜하다."

**결론: 방향은 타당하나 두 가지로 정교화가 필요하다.** LLM 에게 정제는 "덜 중요"가
아니라 **trade-off**이고, 차이의 근본 기제는 "스스로 정제"보다 **차원·표본효율·집계
redundancy** 에 있다.

## 1) 타당한 부분 — BoW/통계 모델은 입력 잡음에 더 취약
- TF-IDF+LogReg 는 **어휘 차원마다 독립 가중치**를 둔다. idf 와 정규화가 무관 토큰을
  부분적으로 누르지만, **소표본**에선 수만 토큰 중 무엇이 유의한지 신뢰성 있게 추정하기
  어렵다 → 유통·자동차·국제일반 등 무관 기사가 더하는 **잡음 차원이 과적합을 키운다.**
  사전 필터는 이 차원을 직접 제거하므로 일관된 개선이 나온다.
- 반면 RoBERTa 는 **두 겹의 내부 가중** 기제를 갖는다: (a) 트랜스포머 self-attention
  (헤드라인 내 단어의 문맥적 중요도), (b) 본 모델의 learnable-query attention pooling
  (헤드라인 **간** 중요도). EXP-D 에서 top-1 lift 가 1.07×→**7.85×** 로 커지며 실제로
  특정 헤드라인에 집중하도록 학습됐다 → 잡음을 일부 자체적으로 하향한다. 그래서 사전
  필터의 한계효용이 작다. → **해석의 핵심 직관은 옳다.**

## 2) 정교화 ① — LLM 에게 정제는 "덜 중요"가 아니라 **trade-off**
- 필터는 잡음(−)만 줄이는 게 아니라 **볼륨·중복(redundancy)(−)** 도 함께 줄인다.
  RoBERTa 는 **약한 신호를 다수 헤드라인에 걸쳐 집계**해 강건해지는데, market 필터가
  일별 기사를 **431 → 42건**으로 줄이고(아래 표) 9% 의 날은 pooling 입력이 30건 미만이
  되자 성능이 하락했다(KOSPI h1 0.318→0.226). 반대로 macro(98.5건/일, 30건 미만 0%)는
  RoBERTa 에 덜 해롭고 일부 셀은 오히려 개선(KOSDAQ h1 0.293→0.327).

  | multiyear 입력 | 일평균 헤드라인 | 30건 미만 비율 |
  |---|---|---|
  | all | 430.9 | 0% |
  | macro | 98.5 | 0% |
  | market | 41.6 | 9% |

- 즉 LLM 은 noise 를 견디는 대신 **redundancy 를 활용**하므로, 필터는 중립이 아니라
  **음(−)** 일 수 있다. 정확한 진술은 "정제 중요도가 덜하다"가 아니라
  **"LLM 에겐 정제가 noise↓ vs volume↓ 의 trade-off 이며, 본 데이터에선 net 중립~음"**.

## 3) 정교화 ② — 근본 기제는 "스스로 정제"보다 **차원·표본효율**
- **BoW(TF-IDF)**: 희소 **고차원**(수만 차원), 학습 가중치 추정이 데이터량에 비례해
  필요 → 잡음 차원이 과적합에 직결. **정제 = 차원 축소 = 표본효율↑** → 소표본일수록
  필터 이득이 큼(EXP-C 가 2024 소표본에서 특히 크게 개선된 이유).
- **LLM(RoBERTa)**: 사전학습된 **밀집·고정차원** 표현이라 무관 토큰이 별도 차원을 만들지
  않는다(잡음 민감도↓). 대신 충분한 **데이터·아이템 수**가 있어야 약신호 집계가 작동한다
  — EXP-D 에서 데이터를 3년으로 늘리자 붕괴가 해소된 것이 이를 입증. 요컨대
  **BoW 의 낮은 표본효율을 '정제'로 보완하느냐, LLM 의 데이터·볼륨 의존을 '확대'로
  충족하느냐**의 차이다.

## 4) 증거의 강도와 한계 (정직한 caveat)
- 단일 데이터셋·약신호·시드 1개, 효과크기 0.02–0.08 수준 → **"경향"이지 정설이 아니다.**
- **소비 방식의 비대칭**이 결과를 일부 혼입한다: TF-IDF 는 하루 **전체** 헤드라인을 한
  document 로, RoBERTa 는 **최신 30건**만 소비한다. 따라서 "필터가 TF-IDF 를 더 돕는다"
  에는 'TF-IDF 가 잡음을 더 많이 들이마신다'는 소비 차이가 섞여 있다. 동일 입력 집합으로
  통제하면(예: 두 모델 모두 동일 상위 N) 더 깨끗한 비교가 된다(→ 향후 실험 후보).

## 5) 보고서 메시지 (요약)
> **전처리·정제의 가치는 모델 종류에 의존한다.** 통계·BoW 계열에는 정제가 큰 레버
> (저차원화·과적합 억제)지만, 사전학습 LLM 에는 데이터·볼륨이 더 큰 레버이며 정제는
> noise↓·volume↓ 의 trade-off 다. 실무 지침: **저자원·BoW → 정제 우선**,
> **충분한 데이터·LLM → 볼륨/문맥 보존**. 단 본 과제에선 어느 쪽도 무작위(0.333)를 크게
> 넘지 못해, 헤드라인의 단기 방향 예측력은 본질적으로 약하다는 주 결론은 유지된다.
