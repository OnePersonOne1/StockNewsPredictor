# 인수인계 — 기말 보고서 작성용 (웹 Claude 대상, 자기완결 문서)

> **목적**: 이 한 파일만 읽고도 StockNewsPredictor 프로젝트의 학부 기말 보고서(과제 대체)를
> 작성할 수 있도록, 연구질문·데이터·방법론·전체 실험(EXP-A~X) 결과·서사·한계를 모두 담았다.
> 레포 파일을 못 보는 환경(웹)을 가정하므로 수치는 본문에 직접 적었다. **작성 톤: 성능 자랑이
> 아니라 "과정·해석·정직한 보고"가 핵심**(지도 의도). 음(부정)·실패 실험도 원인과 함께 서술.
> 작성일 기준: 2026-06-10. 보고서 언어: 한국어.

## 0. 한 줄 요약 / 연구질문
- **연구질문**: 뉴스 헤드라인의 주가 *방향* 예측력은 얼마나 먼 미래(horizon)까지, 어떤 조건에서
  유효한가? 그리고 그 신호를 **어떤 평가 지표로 보느냐가 결론을 바꾸는가?**
- **대상**: KOSPI·KOSDAQ 지수(주), 삼성전자 단일종목(확장). horizon h ∈ {1,5,21,252} 거래일.
- **한 줄 결론**: 헤드라인의 방향 예측력은 **약하지만 0은 아니다**. 단 3-class macro-F1 로는
  그 약신호가 가려지고, **이진 방향·IC(정보계수)로 봐야 드러난다**. 탐지는 되나 **거래 가능한
  알파는 아니다**(약형 시장효율 가설과 정합). 저자원에서 딥러닝이 무너지는 건 2025 같은 특정
  시기 탓이 아니라 **데이터 크기** 탓이고, 입력은 **적고·최신·시각순**일수록 좋다.

## 1. 데이터
- **헤드라인(본체)**: BIGKinds(한국언론진흥재단) export. 한국경제·조선일보·한겨레의 경제·국제
  기사 제목. 2021~2024년. 2024년 단독 104,920건(일평균 ~287). **제목만 사용**(본문 추가는
  무효 — EXP-K). 저작권 콘텐츠라 **학술 비영리 재현 목적**으로만 사용.
- **추가 자료**: IT·과학 섹션 별도 export(EXP-L/N), 삼성전자 단독 검색식 export 2021~2025
  (24,357건, 일평균 ~13건 — EXP-V/W/X).
- **가격**: FinanceDataReader(자격증명 불필요). 지수 KS11/KQ11, 삼성전자 005930. 2021~2026.6.
  KRX 종가와 일치 확인.
- **라벨**: forward return r_h 에 ternary {-1,0,+1}, 임계 ε_h = 0.3·σ_h. **σ_h 는 train 에서만**
  계산(누설 방지). 방향 신호 검정 시엔 **이진 up/down**(r_h>0)으로 전환해 IC 로 평가.

## 2. 방법론 (고정 설계 — 전 실험 불변)
- **Temporal split**(랜덤 분할 금지). **Look-ahead 봉쇄**: 뉴스일 D → 첫 거래일 T(T>D 엄격),
  입력 헤드라인은 T 장 시작 이전 발행분만. forward return 미존재 시 -100 ignore.
- **모델(RoBERTa)**: `klue/roberta-base` full fine-tune. 하루치 헤드라인 각각을 인코딩 후
  **단일 learnable query attention pooling** 으로 하루 벡터 1개 생성(→ attention 가중치는
  4 horizon 공통). 그 위 **multi-task 4-horizon 분류 head**. 모델 확대(large) 미사용.
- **입력 절단**: 하루 헤드라인 중 **최신 MAX_HEADLINES(기본 30)개**만 RoBERTa 가 봄. 원래 GPU
  메모리 제약이었으나, 키우면 오히려 성능↓(EXP-P) → 모델 특성으로 판명.
- **기저(baseline)**: ① random(이론 0.333) ② wordcount(비-ML 렉시콘 단어개수) ③ **TF-IDF +
  LogisticRegression**(`min_df=2, max_features=20000, class_weight=balanced`). TF-IDF 는 기본적으로
  하루 전체 헤드라인을 봄(공정비교 시 top-N 으로 제한 — EXP-R).
- **평가 지표**: 3-class **macro-F1**(주), 그리고 **이진 방향 정확도 / IC(Spearman, score vs
  return) / 롱숏 백테스트 + 유의성**(EXP-H 이후). 이 지표 선택이 결론을 가른다(핵심 메시지).
- 공통 하이퍼파라미터: AdamW, lr 2e-5, wd 0.01, warmup 0.1, grad_clip 1.0, best-val 체크포인트.
  epochs 는 초기 4 → 대표본엔 부족 판명(EXP-T) 후 다년·삼성은 30.

## 4. 보고서가 풀어야 할 **서사 아크**(핵심 5)

1. **"무엇도 무작위를 못 넘으면 의미 없는 것 아닌가?"라는 의심에서 출발** → 평가 지표를 바꾸자
   (macro-F1 → 이진 IC) **약하지만 실재하는 방향 신호**가 드러났다(EXP-H). 이것이 프로젝트의
   방법론적 전환점. **지표 선택이 결론을 만든다**가 1번 메시지.
   - 단, 정직하게: 세 지표(이진/IC/백테스트)는 같은 score·수익에서 파생돼 **독립 증거가 아님**.
     방어 가능한 근거는 셀별 Spearman p + 시드 견고성. 롱숏 백테스트는 **비유의** → 거래 불가.
2. **저자원 딥러닝 붕괴의 원인 규명**: 406행(2024 단년)에선 예측·attention 모두 붕괴 →
   3년(1358행)에서 해소(attention top-1 lift 1.07×→7.85×). **정상기 소표본(2021/22/23)도 모두
   붕괴**(EXP-F) → "2025가 이상해서"가 아니라 **데이터 크기**가 원인(EXP-M 이 다중시드로 확정).
3. **입력은 적고·최신·시각순이 최적**: 관련성 필터는 TF-IDF 를 키우고(EXP-C/E), 신호는 'IT 주제'가
   아니라 **경제적 프레이밍**에 있다(EXP-N). 헤드라인을 더 넣으면 RoBERTa↓(EXP-P), 랜덤 셔플도
   손해(EXP-Q), 버려졌던 **뉴스 식별자 실시각으로 정렬하면 최선**(EXP-S). 본문 추가 무효(EXP-K).
   → **RoBERTa 는 적고 시의성 있는 입력, TF-IDF 는 많은 입력**을 선호(모델 특성, 핸디캡 아님).
4. **방법론 자기검증**: 4 epoch 이 대표본엔 부족했음을 뒤늦게 발견(EXP-T) → 기존 다년 결과는
   '보수적'. RoBERTa 단일 시드 변동 ±0.04+ 라 미세 비교는 TF-IDF(결정적)·다중시드·경향에 의존.
5. **단일종목·분포 이동 이야기**: 삼성 단독으로 확장(EXP-V)했더니 2025 OOS 약함 →
   **원인은 2025 의 유례없는 +124.5% 폭등장**(EXP-W). 2021–23(하락 우세)으로 학습한 모델이
   2025 에 up 을 33%만 예측(실제 62%)해 체계적으로 틀린다(분포 이동). **사용자가 "2024 수치는
   val 선택으로 낙관적일 수 있다"고 지적** → 2024 를 완전 held-out 으로 재검(EXP-X): 2024 우위는
   0.59→0.52 로 **일부 낙관편향이 맞았으나** '항상 up'(0.43)보단 위로 잔존, **2025 실패 결론은
   설계와 무관하게 견고**. → 정직한 자기수정의 사례로 강조할 것.

## 6. 한계·정직한 보고 포인트 (반드시 포함)
- 세 방향 지표는 **상관된 뷰**이지 3배 증거가 아니다. 과거 "24/24 ⇒ p≈6e-8" 주장은 **철회**함.
- RoBERTa 단일 셀 수치는 시드 변동 ±0.04+ → 미세 우열은 단정 금지.
- 삼성 EXP-V~X 는 단일 시드·단일 종목·단년 레짐 의존. h21/h252 는 2025 forward 가 2026+ 가격을
  요구해 NaN 多(사실상 h1/h5 만 유효).
- EXP-X 의 2024 하락(0.59→0.52)은 val-선택 효과와 학습데이터 1년 감소가 **교란**돼 완전 분리 불가.
- 헤드라인 제목은 저작권물 → 데이터 재배포 금지, 재현은 환경변수/검색식으로.

## 7. 최종 실험(EXP-U) — **⚠ 시간상 아직 진행 중(미완료)**
- **정의**: 지수 multiyear, **하루 헤드라인을 (top-30 절단 없이) 대폭 늘려**(큰 MAX_HEADLINES,
  GRAD_CKPT 메모리 절감) **이진 × 30 epoch** 로 풀 재학습. EXP-P(헤드라인 많으면 RoBERTa↓)와
  EXP-T(4 epoch 부족)를 **충분 학습·전량 입력** 조건에서 동시 재검하는 마지막 통제 실험.
- **확정·착수된 설정(2026-06-10 15:24 KST 시작)**: `EXP_PROFILE=multiyear HEADLINE_ORDER=time
  BINARY=1 GRAD_CKPT=1`, **MAX_HEADLINES=1059**(일별 최대치 = 전체 헤드라인 100% 커버, top-30
  절단 없음 — 평균 430/일, 최대 1059/일), **batch=2**(GPU 17.8/24.5GB 적합, OOM 없음), **30 epoch**,
  train 1358 / val 120 / test 488. 산출: `phase2/results_multiyear_ordtime_bin/`(trainlog 자동 저장).
- **소요 추정**: 삼성 mh64/30ep ≈17분 기준 환산 시 입력량 약 16×·데이터 2.75× → **약 1~2일**.
- **현재 상태**: **학습 진행 중(미완료)**. 보고서에는 **"설계 확정·학습 진행 중이라 결과 미포함,
  추후 보유"**로 명시할 것. 본 보고서의 결론은 EXP-U 에 의존하지 않는다(EXP-P/T 가 이미 방향성을
  보였고 EXP-U 는 충분학습·전량입력 조건의 확인·강화 목적).
- 예상 결과(가설, 보고서엔 '예상'으로): 입력을 늘려도 RoBERTa 의 방향 IC 는 개선되지 않거나
  소폭 악화(EXP-P 연장), 30 epoch 로 다년 수치는 EXP-T 만큼 소폭 상승할 것.
