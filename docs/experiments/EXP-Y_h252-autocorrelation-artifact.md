# EXP-Y: h=252 macro-F1(0.646)이 예측력인가 — 라벨 자기상관 아티팩트 검증

| 항목 | 내용 |
|---|---|
| 상태 | 완료 |
| 날짜 | 2026-06-11 |
| 커밋 | `<this>` |
| 데이터 | `samsung_cv` 프로필 binary (`results_samsung_cv_ordtime_bin`), test=2024–2025 |
| 산출물 | `phase2/shuffle_validation.py`, 본 문서 |

## 1. 동기 / 가설
`results_samsung_cv_ordtime_bin/figures/horizon_decay.png` 에서 binary 모델의 test macro-F1 이
h=1/5/21 은 **0.537 / 0.503 / 0.497 (≈random 0.5)** 인데 **h=252 만 0.646 으로 튄다.**
horizon 이 길수록 인과는 약해야 정상인데 가장 긴 horizon 만 높다는 것은 진짜 예측력이라기보다
**라벨 구조의 아티팩트**를 의심하게 한다.

> **가설:** h=252 의 0.646 은 헤드라인의 1년 예측력이 아니라, 252거래일(≈1년) 라벨이 거의
> 단일 전환(블록 2개)이라 생기는 **시간 자기상관**의 산물이다.

선행 관찰(`metrics.json` `note_h252`, 그리고 라벨 run-length 분석): h=252 의 337개 유효 test
라벨은 시간순으로 **딱 2블록** — 앞 163일 전부 down, 뒤 174일 전부 up (2025 +124% 급등 구간).
대조로 h=1 은 250블록(평균 런 1.9), h=5/21 은 96/58블록. → h=252 의 **유효 독립 표본 ≈ 2개.**

## 2. 설정 — `phase2/shuffle_validation.py`
고정 설계(klue/roberta-base, samsung_cv split, mh64, binary, `BEST_CKPT`) 그대로, 학습된
모델을 test 에 다음 3가지로 평가:

- **(A) REAL** — 실제 test 평가로 horizon별 macro-F1 재현.
- **(B) SHUFFLE** — test 안에서 `headlines` 를 **행 간 무작위 셔플**(날짜·라벨 고정, 10 seed).
  점수가 유지되면 위치/상수 의존, 0.5로 내려가면 헤드라인 **내용**에 의존.
- **(C) SHIFT-NULL** — 실제 예측을 고정하고 **라벨 시퀀스를 원형 이동**(circular shift)시켜
  macro-F1 분포를 만든 permutation 검정. 두 시계열의 자기상관을 **보존**한 채 정렬만 바꾸므로,
  "관측된 정렬이 우연 정렬보다 특별한가"를 p-value=P(null≥real) 로 측정.

## 3. 결과

| horizon | (A) REAL | (B) 헤드라인 셔플 (mean±std) | (C) shift-null p | null 최댓값 |
|---|---|---|---|---|
| 1 | 0.537 | 0.496 ± 0.019 | 0.091 | 0.570 |
| 5 | 0.503 | 0.478 ± 0.023 | 0.254 | 0.540 |
| 21 | 0.497 | 0.491 ± 0.018 | 0.608 | 0.547 |
| **252** | **0.646** | **0.501 ± 0.032** | **0.119** | **0.659** |

REAL h=252 = 0.646 으로 `metrics.json` 과 정확히 일치(검증 파이프라인 정상).

## 4. 분석
**(B) — 헤드라인을 섞으면 h=252 가 0.646 → 0.50 으로 무너진다.** 즉 0.646 은 순수한 위치/상수가
아니라 **헤드라인 내용에 의존**한다. ("그냥 날짜만 보고 찍는다"는 단순 해석은 기각 — 모델은
실제로 헤드라인을 읽어 두 블록을 가른다.)

**(C) — 그러나 그 가름은 통계적으로 유의하지 않다(p≈0.12).** 라벨을 자기상관 보존한 채 이동시킨
null 에서 **11.9% 의 무작위 정렬이 0.646 이상**을 냈고, **null 최댓값 0.659 가 실제 0.646 을
넘는다.** 라벨이 전환 1회뿐이라, 시간적으로 뭉친(blocky) 예측을 아무 단일 경계에 맞추기만 해도
0.65 수준은 흔하다.

**종합 기제:** 모델 예측은 헤드라인의 **시간 드리프트 특징**(해마다 변하는 토픽·어휘)을 타고
시간적으로 뭉쳐 있다 → 셔플하면 무너지고(B), 실제 시점에선 라벨의 단일 전환과 대충 정렬된다.
하지만 그 정렬은 **자기상관만으로 기대되는 수준**이다(C). 그 특징이 진짜 1년 신호라면 라벨이
매일 바뀌는 h=1/5/21 에서 먼저 잡혀야 하나 거기선 전부 ≈0.50. → h=252 가 쓰는 것은 예측 신호가
아니라 **시간 상관 잡음(spurious temporal feature)** 이다.

**모든 horizon 에서 유의한 예측력 없음**(전부 p>0.05). h=1 의 0.537 조차 p=0.091 로 비유의.

## 5. 한계
- shift-null 은 라벨이 1전환뿐이라 null 이 **coarse**(p값은 근사). 그래도 "real 이 null 최댓값
  아래"라는 정성 결론은 견고하다.
- 단일 종목(삼성전자)·단일 test 기간. 일반화 주장 아님 — 본 실험의 주장은 "이 점수는 신뢰 못 함".
- non-cv binary(`results_samsung_ordtime_bin`)는 애초에 h=252=0.392(<0.5)라 인플레 사례 아님 —
  본 검증은 인플레가 나타난 cv 케이스를 겨냥.

## 6. 결론 / 다음 (보고서 메시지)
- **h=252 의 0.646 은 "헤드라인이 1년 뒤를 맞힌다"가 아니라, 라벨이 1회만 바뀌는 구조에서
  자기상관이 만든 신기루다.** 보고서·발표에서 h=252 는 신뢰구간/표본 경고와 함께 빼거나
  caveat 을 달고, 메인 메시지는 **h=1/5/21 의 ≈0.50(예측력 없음)** 으로 간다.
- `horizon_decay.png` 캡션에 "h=252 는 유효 독립표본 ≈2, shift-null p=0.12 로 비유의" 명시.
- 일반 교훈: 겹치는(overlapping) 장기 horizon 라벨에서 macro-F1 을 표본 수만큼 독립으로 세면
  신뢰도가 과대평가된다 → 장기 horizon 지표엔 **블록 단위 유의성 검정**을 동반할 것.
